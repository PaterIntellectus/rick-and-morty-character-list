# rick_and_morty_character_list

_Проект построен исходя из требований Технического Задания пришедшего от компании "Effective Mobile"_
Проект подрaзумевает под собой offline-first приложение с просмотром списка персонажей из сериала "Rick and Morty", и возможностью добавления персонажей сериала в список избранных

## Основные фичи проекта:
- Просмотр всех персонажей из сериала "Rick and Morty"
- Возможность создания списка избранных персонажей
- Поддержка тёмной темы


## Инструкция по сборке приложения:
1. Получение кода на своё устройство
- клонировать репозиторий любым из удобных вам способов, например консоль или терминал:
    _git clone https://github.com/PaterIntellectus/rick-and-morty-character-list.git_
- скачать код в архивированном формате и распаковать на устройстве:
    на странице GitHub репозитория с проектом (https://github.com/PaterIntellectus/rick-and-morty-character-list.git) для этого есть большая зелёная кнопка с надписью "Код"/"Code"

2. Получение dart/flutter зависимостей
Для получения зависимостей в корне проекта необходимо открыть консоль/терминал и запустить следующую команду
_dart pub get_

3. Компиляция проекта
- режим отладки приложения:
    _flutter run [--release, --debug]_
- создание исполнительного установщика приложения
    Для android устройств:
    _flutter build apk_
    Для ios устройств:
    _flutter build ipa_


## Инфрастурктурные особенности
Данные о персонажах сериала приходят из общедоступного API сервиса https://rickandmortyapi.com

В этом проекте я впервые для себя попробовал реализовать популярный на стороне Front-End разработки дизайн, под названием Feature Sliced Design (FSD), внутри проекта Flutter

Все запросы к API сервису реализованы через Rest
Кэш и Избранные хранятся в SQLite базе данных
Все приходящие в приложение данные пагинированны

Версия Flutter sdk: ^3.9.2

### Зависимости
- **bloc** - работа с основной "бизнес" логикой приложения
- **bloc_concurency** - удобный трансформаторы для event/state стримов bloc
- **equatable** - удобное создание data классов
- **flutter_bloc** - виджеты для работы с bloc внутри экосистемы Flutter
- **http** - работа с http запросами
- **rxdart** - используется для реализаци стримов на стороне репозитория
- **sqflite** - работа с базой данных SQLite
